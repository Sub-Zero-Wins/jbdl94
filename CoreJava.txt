  the binding or method declaration to method definition happens during run time.

Overriding:
changing parent class implementation of a method in child class.	

  
Rules of overriding
-------------------

1. name and signature must be preserved

2. wrt access modifier

   the visibility of the method can be kept same or can be increased but cannot be decreased.

      public > protected > default > private 

Parent :    public                              protected                     default                         private 






Child  :     public				protected, public            default,protected , public         no inheritance hence no overriding.



not alloed   protected > default > private     private default               private




public = can access anywhere by just importing.

protected= protected is same default + you can access protected members outside package by inheriting the class

default = default aka package level access modifier, i.e members can be accessed only withing the package they are declared in.

private =  private members are accessible only within the class they are declared in.
 

3.  wrt return type.

cannot change return type for primitive return data type.

void int double


for non primitive covariant types can be changes but not contra variant.

4. what all methods do not get overridden.

  private cannot be overridden (reason) private are not inherited

  static cannot be overridden  (reason) overriding is applicable for object level method they exhibit method hiding.

  final cannot be overridden   as the impl declared final.

       final - variable = cannot be re assigned.
       final - method   = cannot be overriden
       final - class  = cannot have a subclass (cannot be inherited)

interfaces.
-----------

an interface is a purely abstract body.

by default all methods in an interface are public and abstract

and all data is public static and final.

implements keyword is used by a class to implement the interface.

one interface can be implemented by many classes

interfaces can perform inheritance including multiple inheritance

a class can inherit another class and implement multiple interfaces at time


class A extends B implements Itr1,Itr2,It3  correct

class A  implements Itr1,Itr2,It3 extends B  wrong


empty interface is called as marker interface

an interface with single abstract method is called as Functional Interface
